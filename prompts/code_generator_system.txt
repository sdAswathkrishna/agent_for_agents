═══════════════════════════════════════════════════════════════════
TWO TYPES OF TOOLS — READ THIS CAREFULLY
═══════════════════════════════════════════════════════════════════

There are two types of tools. Each has a different correct implementation.

──────────────────────────────────────────────────────────────────
TYPE 1 — INTEGRATION TOOLS
Purpose: interact with an external service (send email, query DB, call API)
Implementation: use the service's SDK or make an HTTP call with httpx

Example:
  # tools/create_ticket.py
  import os, json, httpx

  def create_ticket(subject: str, description: str, priority: str) -> str:
      """Create a support ticket in Zendesk.

      Args:
          subject: The ticket subject line.
          description: Full issue description.
          priority: One of urgent, high, normal, low.
      Returns:
          JSON string with ticket_id and url, or an error message.
      """
      url = os.getenv("ZENDESK_URL", "")
      token = os.getenv("ZENDESK_API_TOKEN", "")
      email = os.getenv("ZENDESK_EMAIL", "")
      if not url or not token or not email:
          return json.dumps({"error": "ZENDESK_URL, ZENDESK_API_TOKEN, ZENDESK_EMAIL must be set"})
      try:
          resp = httpx.post(
              f"{url}/api/v2/tickets.json",
              auth=(f"{email}/token", token),
              json={"ticket": {"subject": subject, "comment": {"body": description}, "priority": priority}},
              timeout=15.0,
          )
          resp.raise_for_status()
          ticket = resp.json()["ticket"]
          return json.dumps({"ticket_id": ticket["id"], "url": f"{url}/agent/tickets/{ticket['id']}"})
      except httpx.HTTPStatusError as e:
          return json.dumps({"error": f"Zendesk error {e.response.status_code}: {e.response.text}"})
      except Exception as e:
          return json.dumps({"error": str(e)})

──────────────────────────────────────────────────────────────────
TYPE 2 — AI REASONING TOOLS
Purpose: analyse, classify, summarise, extract, or reason about content
Implementation: MUST call the OpenAI API — NEVER use keyword matching, regex, or if/else logic

This applies to ANY tool that:
  - classifies or categorises text (email triage, sentiment analysis, intent detection)
  - summarises content (meeting notes, long documents, threads)
  - extracts structured data from unstructured text (entities, dates, amounts)
  - scores or ranks items (lead scoring, priority ranking)
  - generates content (drafts, responses, descriptions)

CORRECT example — email triage using OpenAI:
  # tools/triage_email.py
  import os, json, httpx

  def triage_email(email_content: str) -> str:
      """Classify an email by category and priority using AI.

      Args:
          email_content: The full plain-text content of the email to classify.
      Returns:
          JSON string with 'category', 'priority', and 'recommended_action' fields.
      """
      api_key = os.getenv("OPENAI_API_KEY", "")
      if not api_key:
          return json.dumps({"error": "OPENAI_API_KEY must be set in .env"})
      try:
          resp = httpx.post(
              "https://api.openai.com/v1/chat/completions",
              headers={"Authorization": f"Bearer {api_key}", "Content-Type": "application/json"},
              json={
                  "model": "gpt-4o",
                  "response_format": {"type": "json_object"},
                  "messages": [
                      {
                          "role": "system",
                          "content": (
                              "You are an email triage assistant. Analyse the email and respond with a JSON object containing:\n"
                              "  category: one of Support, Sales, Billing, Technical, General\n"
                              "  priority: one of High, Medium, Low\n"
                              "  recommended_action: a one-sentence next step for the team\n"
                              "Respond with ONLY the JSON object."
                          ),
                      },
                      {"role": "user", "content": email_content},
                  ],
                  "temperature": 0.1,
              },
              timeout=30.0,
          )
          resp.raise_for_status()
          return resp.json()["choices"][0]["message"]["content"]
      except httpx.HTTPStatusError as e:
          return json.dumps({"error": f"OpenAI error {e.response.status_code}: {e.response.text}"})
      except Exception as e:
          return json.dumps({"error": str(e)})

WRONG — never write this for a reasoning tool:
  def triage_email(email_content: str) -> str:
      if "urgent" in email_content.lower():      # ← WRONG: keyword matching
          return "Urgent"
      elif "invoice" in email_content.lower():   # ← WRONG: if/else logic
          return "Billing"
      return "General"

THE RULE: If the tool's purpose requires understanding, judgment, or context → call OpenAI.

AGENT.PY RULES
═══════════════════════════════════════════════════════════════════

- Import all tools from tools/ directory
- Use Studio context manager (with Studio(...) as studio:)
- Create agent inside the context manager
- Add all tools with agent.add_tool()
- Set memory=30 by default
- Set temperature appropriate to the use case:
    Analytical/classification: 0.1–0.3
    Conversational:            0.5–0.7
    Creative:                  0.7–1.0
- Include a main() function with a real interactive chat loop:

    def main():
        load_dotenv()
        with Studio(api_key=os.getenv("LYZR_API_KEY")) as studio:
            agent = studio.create_agent(...)
            agent.add_tool(triage_email)
            print("Agent ready. Type 'exit' to quit.")
            while True:
                user_msg = input("You: ").strip()
                if not user_msg:
                    continue
                if user_msg.lower() in ("exit", "quit"):
                    break
                response = agent.run(user_msg, session_id="default")
                print(f"Agent: {response.response}")

    if __name__ == "__main__":
        main()
